# Plan: Loader + Payload Integration with claude-setup

## Context

Three repos need to work together end-to-end:

| Repo | Role |
|---|---|
| `claude-setup` | Wizard that generates `loader.zip` and bootstraps repos |
| `claude-payload` | Cloned by the loader at runtime; provisions the environment |
| `loader.zip` | Thin skill uploaded to Claude; clones payload and runs it |

**Current problem:** `claude-payload/src/main.py` hardcodes the owner username
(`0x038b5c`) throughout. The loader templates generated by `claude-setup` already
pass `github_username` as a positional arg to payload, but payload ignores it.
The system only works for `0x038b5c` right now.

**Secondary:** The live loader skill (`/mnt/skills/user/loader/src/main.py`) is an
old manual version — no `github_username` arg, hardcoded payload repo. It predates
`claude-setup` and won't be regenerated until the wizard runs. The plan below fixes
the templates so any newly-generated loader works; the live skill is replaced when
the user re-runs `claude-setup`.

## What Needs to Change

### 1. `claude-payload/src/main.py` — accept `github_username` arg

**Problem:** Three hardcoded strings:
```python
run("git clone https://github.com/0x038b5c/claude-secrets /opt/secrets --depth 1")
run("git clone https://github.com/0x038b5c/claude-state /opt/state --depth 1")
print("GitHub username: 0x038b5c")
```

**Fix:** Read `github_username` from `sys.argv[1]` and derive all URLs from it.

```python
import sys
github_username = sys.argv[1]
run(f"git clone https://github.com/{github_username}/claude-secrets /opt/secrets --depth 1")
run(f"git clone https://github.com/{github_username}/claude-state /opt/state --depth 1")
print(f"GitHub username: {github_username}")
```

Also: `tool.py` hardcodes `0x038b5c` in `project new`. Fix the same way — read
from a config file that `main.py` writes during provisioning.

### 2. `claude-payload/src/tool.py` — parameterize github_username

**Problem:**
```python
run(f"gh repo create {name} --public")
run(f"git remote add origin https://github.com/0x038b5c/{name}.git", ...)
state["project"]["repo"] = f"https://github.com/0x038b5c/{name}.git"
```

**Fix:** Write `github_username` to `/opt/github_username` during `main.py`
provisioning, then read it in `tool.py`:

```python
GITHUB_USERNAME = Path("/opt/github_username").read_text().strip()
```

### 3. `claude-payload` — add `.gitignore`

No `.gitignore` exists. Add one to prevent accidental commits of `.pyc`, 
`__pycache__`, etc.

### 4. `claude-setup` — update `CLAUDE.md` and `.claude/todo.md`

Document the project structure and keep notes current.

## Execution Steps

### Step A — Parameterize `claude-payload/src/main.py`

1. Add `import sys` 
2. Read `github_username = sys.argv[1]`
3. Replace all three hardcoded `0x038b5c` references with `github_username`
4. After decrypting secrets, write `github_username` to `/opt/github_username`

### Step B — Parameterize `claude-payload/src/tool.py`

1. Read `GITHUB_USERNAME = Path("/opt/github_username").read_text().strip()`
2. Replace hardcoded `0x038b5c` in `gh repo create`, `git remote add`, and
   state JSON

### Step C — Add `.gitignore` to `claude-payload`

Standard Python gitignore (pycache, egg-info, dist, build, venv, etc.)

### Step D — Update `claude-setup/CLAUDE.md`

Document the project structure so future sessions don't need to re-read everything.

### Step E — Commit and push both repos

- `claude-payload`: Steps A, B, C
- `claude-setup`: Step D

## Verification

After executing the plan, manually verify the call chain:

```
loader/src/main.py
  → git clone <payload_repo> /opt/payload
  → python -m src.main <github_username>       # passes username
      → clones <github_username>/claude-secrets
      → clones <github_username>/claude-state
      → writes /opt/github_username
      → tool project new <name>                # reads /opt/github_username
          → gh repo create <name>
          → git remote add origin https://github.com/<github_username>/<name>
```

The system should work identically for any `github_username`, not just `0x038b5c`.
